# 圣农经营智能中枢 Hash 路由版设计规格

## 1. 目标

把当前“一个长滚动文档”改造成“首页 + 8 个节点详情页”的单 HTML 应用。

目标结果：

- 首页负责用四阶段经营闭环建立视觉冲击和阅读动机，不承载节点详情预览。
- 点击任一流程节点后进入该节点详情状态，首页品牌展示和总览英雄区不再占位。
- 节点详情顶部只保留完整流程导航、返回首页和“联系我们”预留入口。
- 每个节点只占一个页面状态；节点内部通过交互标签查看不同维度的信息。
- 上一节点的输出必须明确成为下一节点的输入，使 8 个页面构成连续经营闭环。
- 当前节点完全由 URL Hash 决定，不再依赖滚动位置推断。

## 2. 信息架构

### 2.1 路由

使用单 HTML + Hash 路由：

```text
#home
#node/collection
#node/transport
#node/foundation
#node/aily
#node/decision
#node/milestone
#node/coaching
#node/feedback
```

规则：

- 首次打开且没有 Hash 时进入 `#home`。
- 直接刷新 `#home` 时只显示首页叙事；从详情页返回首页不保留节点预览。
- 点击流程节点立即更新 Hash 和页面状态。
- 浏览器刷新、前进、后退后恢复对应页面。
- 未知 Hash 回退到 `#home`，不显示空白页。
- Hash 是唯一当前节点状态源；不再使用 IntersectionObserver 决定节点高亮。

### 2.2 首页

首页由三部分组成：

1. 品牌头：左侧为缩小的 SN 标识与圣农方案名称，中间并列“首页”和“方案详情”，右侧为“联系我们”。
2. 视觉英雄区：以“让每一个经营异常，都走向可验证的结果”为主叙事，解释这不是聊天机器人，而是企业经营闭环。
3. 四阶段架构总览：经营事实进入、智能调查研判、人工批准与执行、结果回读与校准，并通过动态信号表达前后关系。

首页不得展示 8 节点简化流程、节点卡片目录或选中节点预览。“方案详情”和英雄区主按钮统一进入 `#node/collection`。

### 2.3 节点详情页

详情页不显示首页英雄区和大面积品牌头，只显示：

1. 返回首页。
2. 8 节点流程导航。
3. “联系我们”占位。
4. 当前节点标题、职责说明和本页产物。
5. 节点内部标签。
6. “传给下一页”的交接条件和下一节点按钮。

节点内部统一为四个标签：

- 流程示例。
- 输入与产物。
- 技术落地。
- 验收与边界。

内容较多时只在当前标签内自然纵向滚动，不把一个节点拆成多个主页面。

## 3. 八节点连续关系

| 节点 | 本页核心产物 | 下一页前置条件 |
|---|---|---|
| 信息采集 | 原始证据包 | 数据传输层接收带来源、时间、权限、质量和 hash 的证据 |
| 数据传输 | 可重放的标准证据流 | 决策底座取得可靠、幂等、可追溯的数据输入 |
| 决策底座 | 可查询的企业上下文 | Aily 获得统一对象、口径、规则、例外和 event_id |
| Aily Agent | 决策就绪事件包 | 管理层取得事实、缺口、情景方案、约束和风险 |
| 人工决策 | 带 approval_id 的决定 | 里程碑工作流获得已批准动作、预算和回滚条件 |
| 里程碑门 | 可验证执行状态 | 岗位辅导获得真实节点、责任任务和阻塞信息 |
| 岗位辅导 | 人员完成证据 | 结果校准获得执行证据、学习记录和未解决问题 |
| 结果校准 | 新一轮经营事实 | 返回信息采集层，关闭、升级或重开下一轮事件 |

每页“流程示例”必须使用同一个贯穿案例：门店 29.9 元标准价出现 24.9 元成交信号。案例随页面推进，不在每页重新开始。

## 4. 流程颜色

颜色同时编码阶段，节点文字仍明确写出名称，不能只靠颜色表达：

- 信息收集：节点 01-03 和对应连线使用蓝色。
- 智能研判：节点 03 到节点 04 的连线与节点 04 使用橙色。
- 决策执行：节点 05-07 和对应连线使用紫色。
- 反馈校准：节点 08 与回环提示使用绿色。

节点 03 自身仍属于信息收集，因此保持蓝色；从 03 指向 04 的连接线变为橙色。

## 5. 组件边界

### 5.1 Route Controller

职责：解析 Hash、切换 home/detail 状态、处理前进后退、同步当前节点。

输入：`location.hash`。

输出：`view = home | detail`、`activeNodeId`。

### 5.2 Process Navigator

职责：渲染 8 节点、阶段彩色连线、当前节点和完成路径；触发 Hash 导航。

约束：不读取页面滚动位置；移动端允许导航自身横向滚动，但页面不得横向溢出。

### 5.3 Home Overview

职责：解释总体架构、四阶段运转关系和管理层最终决策边界，并引导进入完整方案。

依赖：四阶段配置，不依赖当前节点状态。

### 5.4 Node Page

职责：只渲染一个节点的标题、产物、标签内容和下一节点交接。

依赖：节点配置表、内容模板、Tab Controller。

### 5.5 Tab Controller

职责：在一个节点页内切换四类信息。

约束：切换不改变主节点 Hash；刷新节点页默认回到“流程示例”。

### 5.6 Contact Placeholder

首页和节点页都保留可见入口。当前只显示“联系我们”，不绑定表单、链接或外部发送动作。

## 6. 动效设计

动效采用 HyperFrames 的“先确定英雄帧，再编排进入过程”和 Remotion 的“显式时间、统一缓动、确定性状态”原则，但网页运行时不引入视频渲染器。

### 6.1 页面状态转换

- 首页进入节点页：总览内容淡出并上移 12px，节点页淡入并上移到位。
- 节点切换：先播放连线传递，抵达后同步更新当前节点、Hash 和内容；旧内容 120ms 淡出，新内容 220ms 淡入。
- 返回首页：首页布局恢复，不展示节点导航或节点预览。
- 不使用滚动劫持，不强制翻页，不等待动画完成才允许操作。

### 6.2 流程导航

- 未到达节点和整条底轨默认使用中性灰，不预先显示阶段颜色。
- 初次进入详情页时只有节点 01 点亮；点击后续节点时，阶段色先从当前节点沿连线传递，抵达后再点亮目标节点并切换内容。
- 节点 01-03 的到达路径使用蓝色，03→04 使用橙色，04→07 使用紫色，07→08 使用绿色。
- 切换节点时只更新受影响的连线和节点，不重播整条流程；01→02 的传递时间约 560ms。
- 节点按钮 hover/focus 使用 160ms 边框与颜色过渡，不缩放布局。

### 6.3 节点内容

- 标题、说明和本页产物按 60ms 间隔进入，总时长不超过 360ms。
- 流程示例的四个步骤按顺序轻量显现，箭头与交接条件最后出现。
- 标签切换使用 180ms 淡入，不做横向滑屏，避免高密度内容产生眩晕。

### 6.4 降级

`prefers-reduced-motion: reduce` 时：

- 关闭平滑滚动、路径绘制和错峰进入。
- 页面和标签立即切换。
- 保留颜色、文字和 `aria-current` 状态。

## 7. 数据与内容组织

使用一个版本化节点配置对象作为页面结构源：

```text
node_id
route_label
phase
title
summary
input
process
output
next_node_id
next_condition
tabs.flow
tabs.contract
tabs.technology
tabs.acceptance
```

现有 HTML 中已经验证的业务内容迁移到该配置或对应模板，禁止为了适配新布局删减事实、边界、来源和验收结论。

## 8. 异常与边界处理

- 未知节点：回到首页并保持页面可用。
- 缺失标签内容：显示明确的“内容待补充”占位，不显示空白区域。
- 超长表格：只允许表格容器横向滚动。
- 外部字体失败：回退到系统中文字体和等宽字体。
- JavaScript 不可用：HTML 至少显示首页、完整流程和八节点目录链接。
- “联系我们”在没有真实目标前不得提交、收集或发送任何信息。

## 9. 验收标准

### 路由

- `#home` 与 8 个节点 Hash 均能直接打开。
- 点击、刷新、前进和后退后标题、内容、流程高亮完全一致。
- 进入 Aily 时必须高亮节点 04，不允许停留在节点 03。

### 页面结构

- 首页显示品牌、视觉英雄区和四阶段经营闭环，不显示八节点导航或节点预览。
- 节点详情隐藏首页英雄区，只保留流程导航、返回首页和联系入口。
- 任一时刻只显示一个主节点详情。
- 四个内部标签均可键盘操作。

### 连续闭环

- 每一页明确显示本页产物和下一页前置条件。
- “进入下一节点”按钮进入正确 Hash。
- 节点 08 明确回到节点 01，形成闭环。

### 视觉与动效

- 四阶段颜色映射稳定一致。
- 1440、1024、768、390、320px 无页面横向溢出。
- 动效不引起布局位移或内容重叠。
- 减弱动效模式下无非必要动画。

### 质量

- 浏览器控制台 0 error。
- 当前业务内容、Aily 查询循环、人工审批边界、里程碑质量门和岗位追问能力均保留。
- 原始正式蓝图文件不被覆盖；继续在滚动叙事副本上迭代。

## 10. 明确不做

- 不创建 9 个独立 HTML 文件。
- 不继续使用 IntersectionObserver 推断主路由。
- 不把所有节点内容同时铺在一个长页面。
- 不实现真实联系表单。
- 不引入第二套 Agent 或改变方案业务架构。
- 不把 Remotion 或 HyperFrames 视频播放器嵌入生产网页。
