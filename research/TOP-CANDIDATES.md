## 唯一推荐：圣农集团

官方命题：如果你加入圣农零售铁军，你会如何借助 AI 重新设计“业务员—区域—集团”的三级销售作战闭环，并量化它能带来的业绩增量。

官方详情：<https://activity.feishu.cn/future-talent?detail=shengnongjituan>

### 官网已验证事实

- 圣农零售渠道为百亿级，覆盖 10 大区域、300 多名业务员及多个渠道。
- 业务员日报、经销商台账、POS 流水和价格巡查记录分散在 6 个系统中。
- 价格异常从投诉到处置平均需要 4-5 天。
- 一线客户问题约 70% 重复，新人成材周期超过 6 个月。
- 官网明确要求使用飞书多维表格、妙记、字段捷径和 IM 机器人，设计“AI 驱动的零售销售作战中枢”。
- 官网明确描述从业务员前端问数、区域价格预警到集团爆品决策的全链路，并要求形成可被 12 大区复用的标准化方案。

### 为什么它比“题面写了 FDE”的来菜更适合作为主命题

1. **系统对象明确**：6 套现有系统与三级组织是真实的重塑对象，不需要自行虚构“原系统有什么问题”。
2. **Agent 节点明确**：问数、价格异常、知识支持、区域决策、集团复盘分别需要不同状态、权限和工具。
3. **人机责任明确**：AI 可以发现异常和提出建议，区域负责人仍负责价格与资源动作，集团负责策略和规则。
4. **Harness Loop 必要**：一次异常需要经历观察、诊断、提案、审批、执行、验证和复盘；普通问答或 RAG 不能完成闭环。
5. **业务指标明确**：可以围绕预警时延、问题首次解决率、新人成材周期、任务关闭率和销售增量设计验证。
6. **单人 Demo 可做**：可以用模拟的 POS、价格、日报和经销商数据，完成一条代表性 E2E，不依赖硬件或专业算法。

### 建议的代表性 E2E（方案推导，非官网原文）

```text
POS/价格/业务员日报出现异常
-> 观察 Agent 发现区域价格偏差
-> 诊断 Agent 汇总上下文和可能原因
-> 策略 Agent 形成处理建议与影响范围
-> 飞书向区域负责人发起责任确认
-> 执行 Agent 创建任务并跟踪动作
-> 验证 Agent 比较处理前后销量、价格和库存
-> 复盘 Agent 更新知识与规则
```

这条链路既能展示开源 Agent Harness 的可靠循环，也能展示飞书中的人类责任门禁；ERP/POS/经销商系统以模拟适配器代表，避免伪造真实企业接口。

## 思想匹配度第一：来菜

官方详情：<https://activity.feishu.cn/future-talent?detail=laicaihubeitoupaioutang>

来菜是 108 题中唯一直接把参赛者角色写成 `AI PM/FDE` 的命题，并且明确提出：

- 总部与门店数据割裂；
- 经营指标依赖经验；
- 不同区域和节令的备餐供需难预测；
- 品控依赖总厨和督导经验；
- 顾客反馈分散在问卷、社媒和点评；
- 需要飞书 AI、三方平台和 Agent 构建连锁餐饮经营大脑。

它与长期产品愿景的语言和业务范围最一致，但当前有三项风险：

1. 官网同时要求探索经营预测、销量预测、AI 数字人试菜和门店评估，单人 Demo 容易摊得过大。
2. 预测与数字人试菜可能把重点带向模型效果，而不是管理流程和 Harness 可靠性。
3. 官网不同字段中的门店与会员数字存在不一致：正文出现 34 家/400 万，亮点出现 41+ 家/450+ 万。正式开题材料应避免把这些数字当作无争议事实，或在入围后向企业确认。

因此，来菜是**思想基准题**，但不是当前综合推荐第一。

## 稳健原型题：野人先生

官方详情：<https://activity.feishu.cn/future-talent?detail=yerenxiansheng>

门店、维修部、设备供应商三方工单不同步，设备档案和配件资料不一致，故障状态和责任交接不可见。它非常适合用状态机、Agent 路由、人工确认和失败恢复做出一个完整闭环。

放弃它作为主命题的原因不是难做，而是范围偏窄。它能证明 Harness Loop 的工程价值，但不如圣农充分体现跨系统、跨层级组织和经营决策的系统级重塑。

## 其他高分题为何未选

- **洋葱学园**：跨部门协同非常贴合飞书，但容易做成会议摘要、待办提取和风险提醒，必须增加决策状态和业务门禁才能达到“重塑”。
- **明珞股份**：采购到签收的端到端链路优秀，但报关、船运、ETA 和企业接口的真实性需要后续业务资料支持。
- **钱海网络**：官网直接提出每环节 AI 节点、AtoA 和 MCP 对 MCP，是技术架构最贴合者；但支付失败、拒付、退款和对账诊断存在明显领域与合规门槛。
- **歌尔股份**：问数、预警、推送、任务和复盘链路清楚，但多源数据质量和动态看板工程量较大，也容易退化为 BI Copilot。
- **神州数码**：AI for Process 和快速 Demo 很像 FDE 工具，但它是在生成其他客户的解决方案，缺少一个首先被验证的具体企业流程，存在过早平台化风险。
- **欧派家居**：最接近 ERP/CRM 全价值链重塑，但官网未指定飞书、模块、数据、目标指标或原型边界，单人开发难以证明完整链路。
- **四维图新**：企业数据 Agent 与组织知识方向贴合，但公开目标主要是“3 秒回答”，缺少从洞察到行动、责任和结果验证的闭环，容易成为 RAG 问答。

## 证据边界

- 上述“官网已验证事实”来自官网运行时 `topics` 数据。
- 多数重点题没有单独的“交付内容”字段；文中使用“官网明确要求”或“目标性产出”，不将方案推导伪称为企业官方交付清单。
- 代表性 E2E、Agent 架构和指标设计均为本研究的方案推导，证据状态为 `inferred`。
