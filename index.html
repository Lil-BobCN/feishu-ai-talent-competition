逐源接入矩阵

频率是终局目标建议；圣农具体接口、字段和权限仍需 IT 验收。

源 / 权威信息采集数据传输失败降级
SAP / 正式交易订单、发票、客户/产品主数据、成本毛利、批准价盘。优先 OData/CDS/BAPI/IDoc 或现有集成；AnyCross 编排 API，不直读生产表。保留最近快照并标 stale；价盘或成本过期时禁止高风险建议。
POS / 平台消费者成交、券前/券后、平台/商家/厂家承担方、实付、门店 SKU 库存。零售商授权 API/Webhook；无 API 用 SFTP/对象存储 CSV + manifest/hash。缺报不等于零销量；按覆盖率标 unknown，转人工巡店或补录。
CRM / 业务员客户归属、报价、拜访、活动申请、渠道政策、订单草稿。开放 API/Webhook；飞书表单和审批回调进入事件流。事件重放和幂等；缺失拜访不自动判定未执行。
WMS / 物理库存批次、可用/冻结/临期库存、出入库、发运。API/事件；高量场景才考虑受控 CDC 或只读副本。最后成功快照 + 时间戳；SAP/WMS 差异生成数据质量事件。
经销商 / 上报证据sell-in、sell-out、库存、库龄、退货、回款、促销执行。商城/CRM API 或 EDI；其次 SFTP/模板；最后飞书移动补录。缺报标 unknown，不补零；抽样核对发货、回款和门店 POS。
飞书 / 人采事实巡店照片、价签、陈列、语音、妙记、审批和任务状态。开放平台 API、事件订阅、审批回调；原件与抽取字段分存。Webhook 重试、游标和幂等；模糊照片进入待补证据。
外部市场 / 信号竞品可比价、促销覆盖、有货、行业供需、政策/天气/节日。优先授权 API/正式数据文件；公共网页只依法采集并留快照。多来源交叉；来源失效标 stale，不把缺失当作涨跌。

传输方式选择规则

API / Webhook：默认首选

业务对象有稳定接口合同、需要分钟级查询或回写时使用；低延迟且可做权限和幂等。

AnyCross：集成编排层

负责连接器、字段转换、重试和内网代理，不替代事实库、语义层或高频 POS 流计算。

CDC：仅用于受控高量复制

源为授权关系库、API 不可用且需要增量复制时使用；不能未经 SAP Basis 许可直连生产表。

文件 / SFTP：遗留和外部交换

配合 manifest、文件 hash、行数/金额校验、迟到文件处理和幂等键，慢但可审计。

内网代理：安全通道，不是 Agent

只允许受控出站访问，使用 mTLS/OAuth、IP allowlist、限速与断点缓存。

01 / Prompt 分层

Custom Agent 与 Workflow Agent 节点分别配置，但共享同一版本化证据规范。

Custom Agent System Prompt

角色：圣农经营调查与决策助手。
目标：调查事实、验证原因、组织方案、提交人工审批。
硬规则：
1. 先读 event_id / 语义版本 / 质量状态 / 权限范围。
2. 只调用授权工具；缺失数据必须标 unknown。
3. 先列原因假设，再逐项取证；所有结论引用来源与时间。
4. 阈值、金额、预测调用确定性服务，不由模型心算。
5. 价格、存栏、产能、市场进入不得自动拍板。
6. 未有 approval_id 不得写回；不得把模型记忆当企业事实。
Workflow Agent Node Prompt

输入：event_id、event_type、initial_signal、semantic_version。
步骤：
1. 调用 get_event_context；质量失败则输出 data_issue。
2. 生成互斥原因假设和证据需求。
3. 在允许的领域 MCP 内取证，最多按配置迭代。
4. 调用 simulate_scenarios 获取确定性影响区间。
5. 按 decision_packet_schema 输出 JSON。
6. 不调用正式写入工具，不等待人工审批，不维持跨天会话。

怎样证明 Aily 真能承担唯一 Agent

验收对象不是“机器人能聊天”，而是同一闭环在真实数据、失败和审批等待下仍然成立。

E2E HARD GATE
01 / 证据真实性 MUST PASS

29.9→24.9 合法券、未授权低价和规格差异三种样本均能正确区分；每个结论带 source_id/time/schema。

02 / 缺失数据处理 MUST PASS

接口缺失、数据过期或来源冲突时明确输出 unknown/data_issue，不补零、不编造、不继续高风险建议。

03 / 确定性计算 100%

实付价、毛利、库存、预算和预测字段与独立 oracle 结果一致；Agent 只解释，不自行心算。

04 / 未审批不写入 0 VIOLATION

缺少 approval_id、decision_version 或权限不符时，所有改价、存栏、产能和市场动作被服务端拒绝。

05 / 跨天恢复 MUST PASS

审批等待 24 小时后按 event_id 恢复，不依赖原 Agent Run 持续在线；不会重复建案或重复下发任务。

06 / 故障与幂等 MUST PASS

重复、乱序、连接器失败和超时注入后可重试/补偿；失败状态不能伪装成已完成。

07 / 多区域战略风险 MUST PASS

六区域降价滞销时生成多个原因假设，并调用模型比较促销、减产、存栏和海外方案，不直接断言需求下降。

08 / 权限与审计 MUST PASS

用户、服务和共享身份访问范围正确；工具调用、Prompt/Skill/规则/模型版本和审批结果可追溯。

09 / 里程碑质量门 0 EARLY ALERT

货物仍在运输或门店登记时不得误报未上架；达到正式节点并越过宽限期后才执行对应质量检查。

10 / 岗位辅导 SOURCE-BOUND

新人追问能返回当前岗位、任务和里程碑相关指导；所有职责、步骤和案例带来源与版本，不越过责任边界。

10 MIN DEFAULTWorkflow Agent 节点默认超时；最高值需租户核验。
20 ≠ CONCURRENCY公开口径是每人最多配置 20 个定时任务，不是执行并发或应用容量。
SESSION ≠ EVENTAily 会话/Run 不是跨天经营事件状态机。
OPENAPI AVAILABLE自定义智能体 OpenAPI 已发布；圣农租户开通、同租户可见范围仍需验收。
最终判断标准
Aily 通过上述生产验收，则不增加外部 Agent。若问题出在接口、语义、计算或状态，先补齐确定性服务；只有 Agent 运行时确实无法满足长任务恢复、并发、私有部署或治理要求时，才评估替代运行时。

04 / 价格异常与竞品让利事件

29.9 → 24.9 / 3-DAY RESPONSE

价格事件：从一个异常点逐级调查

POS 暴露门店到手价 24.9，基准价 29.9；脚本只登记可信信号，Agent 决定下一项最小查询。

SIGNAL → DECISION
01

脚本暴露异常字段

POS 报告某 SKU × 门店的实际价低于正式价盘；记录来源、时间、对象和偏差，不判断问题最终有多大。

deterministic detector → signal packet
02

统一事件受理 Skill 启动

Aily 读取当前已知和未知，生成下一项最小、必要、已授权的查询计划。

observe → assess → plan
03

查不清时，才逐级扩大调查

先核验单店券与审批，再扩至区域门店、渠道库存、投诉和竞品，必要时进入集团供需、产能与存栏。

单店 → 区域 → 跨部门 → 集团
04

并行补齐证据

受控 MCP 查询 SAP/POS/CRM/WMS；无法查询的字段自动映射数据责任人，飞书发起补数请求并追踪已读与 SLA。

tool retrieval ∥ human evidence request
05

形成调查结果包

明确已经确认的事实、可能原因、仍缺的信息、调查过程和影响范围，为下一步生成方案提供可靠输入。

调查结果 → 方案生成输入
06

领域 Skill 生成方案

价格、竞品或库存 Skill 调用确定性模型，比较跟价、定向券、规格区隔和不跟价的销量、毛利、库存与预算影响。

domain skill → scenario service
07

形成决策就绪包

把事实、缺口、备选方案、影响、预算、停止条件和恢复办法放进同一份材料，让管理层可以直接比较和审批。

调查结果包 + 方案测算 → 决策就绪包
08

管理层审批与业务节点执行

批准后拆解销售、库存、生产和海外任务；每项任务按真实业务节点推进，达到节点后才启动质量检查。

approval_id → 责任任务 → 业务节点
09

结果验证与经验沉淀

6/24/48/72 小时回读价格、终端销量、库存和投诉；恢复才关闭，并把有效过程沉淀为规则、语义和岗位案例。

复查 → 关闭 / 升级 / 重开

管理层看到的方案包

Agent 给选择，不替管理层选择。

定向券 短期

只覆盖发生竞品冲击且库存健康的门店，控制价格外溢。

规格区隔 中期

通过克重、组合装或渠道专供 SKU 避免全盘降价。

库存优先 临期

在批准的临期窗口内释放库存，明确成本承担和停止条件。

不跟价 守价

保持贡献毛利，持续观察 sell-out 和竞品覆盖，设置升级阈值。

关键决策由管理层把关

价格、存栏、产能和市场进入建议必须有批准人、版本、生效期、预算上限和恢复办法。

外部 Agent 的触发条件

不是默认组件

当前终局不安装第二套 Agent；先用 Aily + 后端服务完成租户 E2E。

只有硬缺口才升级

私有化模型、长任务恢复、复杂补偿、精细治理或生产 SLA 经验证无法满足。

升级方式

外部 Agent 是替代运行时，不与 Aily 并行做同一决策，避免“双脑”状态冲突。
